## 3. Processes 패널이 무엇을 표시하는지 설명
사용: Bash, Read

Toby: Processes는 뭘 나타내는 거지?

Claude:
- 코드 위치: `src/ui.ts:540–556` (렌더), `src/parser.ts:1014–1051` (`loadProcessInfo`).
- 모니터링 중인 Claude Code 세션의 PID를 `~/.claude/sessions/<pid>.json`(activeSessions)에서 찾아, `ps -eo pid,ppid,rss,comm`으로 ppid가 그 PID인 자식 프로세스를 한 단계 깊이만 수집.
- 표시 형식: `pid:1234 mem:42MB <command>`, 메모리 내림차순 정렬, 상위 5개 (내부 보관 10개).
- 자식이 0개면 박스 자체가 안 그려짐.
- 헤더의 `Mem:XXMB`는 Claude Code 부모 프로세스의 RSS, Processes 박스는 그 부모의 자식들 — 별개 데이터.

## 4. 완료된 process도 표시되는지 확인
사용: Bash

Toby: 수행이 완료된 process도 표시되나?

Claude:
- 결론: 표시되지 않음.
- 근거: `ps`는 OS 프로세스 테이블 라이브 스냅샷, exit한 프로세스는 다음 refresh(`REFRESH_INTERVAL=2000`)에서 사라짐. `state.childProcesses`는 매 tick마다 통째로 덮어써지고 (parser.ts:1051) 이전 상태와 머지 안 함.
- 예외: 좀비 프로세스(exit했지만 reap 안 된 상태)는 잠깐 `mem:0MB`로 보일 수 있으나 Claude Code가 즉시 reap하므로 실무상 거의 안 보임.
