## 3. False-positive Ralph Loop 활성화 및 정리
사용: state MCP tools (list_active, clear)

경과: "ralph"와 "ultrawork" 단어가 질문에 포함되자 OMC keyword-detector hook이 자동으로 ralph+ultrawork 모드를 활성화시킴. stop hook이 "The boulder never stops"로 계속 실행 강제. 실제 유저 의도는 정보성 질문이었으므로 false positive.

Claude 정리:
- `state_list_active` → ralph + ultrawork 둘 다 감지
- `state_clear(mode=ralph)`, `state_clear(mode=ultrawork)`, `state_clear(mode=skill-active)` 3개 병렬 클리어
- `state_list_active` 재확인: No modes active
- 방지 방법 안내: `OMC_SKIP_HOOKS=keyword-detector` 또는 `DISABLE_OMC=1`

**재미있는 메타 현상**: 이 질문 자체가 이전 턴에서 추상적으로 설명한 "ralph+ultrawork Combined Mode"의 실증적 재현이 됨.
